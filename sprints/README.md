# Sprints

Sprints track **what we are building and how much is done**. Each sprint is one Markdown file
with a clear goal and a task checklist that gets updated in place as work progresses.

## Convention

- One file per sprint: `sprints/sprint-NN-<short-name>.md`.
- Each sprint has: goal, scope (in/out), tasks (checklist), status, and notes/decisions.
- Update the checklist and status as work moves. The file is a living document.
- Keep sprints small and well-defined. One clear outcome per sprint.

## Status values
`planning` → `in-progress` → `review` → `done`
